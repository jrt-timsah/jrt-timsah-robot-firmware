# JRT Timsah Robot Firmware (CoRE-2 Robot Firmware)

このファームウェアは、CoRE\-2ベースのロボット用に開発されました。ArduinoシールドVer\.2024およびコントローラ基板Ver\.2を使用し、ディスク射出機構を含む基本的なロボット操作を制御します。

---

| 項目 | 詳細 | 備考 |
| :--- | :--- | :--- |
| **対象ロボット** | CoRE\-2ベースロボット | |
| **Arduinoシールド** | Ver\.2024 | |
| **コントローラ基板** | Ver\.2 | |
| **制御周期** | 10 msec (100 Hz) | `period`変数で定義 |
| **通信速度** | 115200 bps | PC、コントローラ、オートレフェリ通信 |
| **射出用モータ** | サーボ制御 (`SHOT`) | ディスク射出動作 |
| **射出用ローラ(BL)** | サーボ信号制御 (`ROLLER`) | ディスク回転用 |

## コントローラ入力割付け (Ver\.2用)

コントローラからの受信データ (`RxData`) に基づく入力割付けです。

| 定義名 | `RxData`ビット/配列 | 機能 | 範囲/状態 |
| :--- | :--- | :--- | :--- |
| `SW_ENABLE` | `RxData[4]` (bit 0) | **操作許可スイッチ** | 1: 操作許可 / 0: 操作禁止 |
| `SW_SHOT` | `RxData[4]` (bit 1) | ディスク射出ボタン | 1: 押されている / 0: 離されている |
| `SW_ROLLER` | `RxData[4]` (bit 2) | ローラー回転ボタン | 1: 押されている / 0: 離されている |
| `AS_LeftX` | `RxData[0]` | 左スティック（左右移動）| \-100 〜 100 |
| `AS_LeftY` | `RxData[1]` | 左スティック (前後移動) | \-100 〜 100 |
| `AS_Right` | `RxData[2]` | 右スティック (回転移動) | \-100 〜 100 |
| `AS_Vol` | `RxData[3]` | ピッチ調整ボリューム | 0 〜 255 |

## 主要な制御機能

### 1. 走行制御 (`Wheel`)

* **左右独立制御:** `MOTOR1~4`が`AS_LeftX`と`AS_LeftY`で前後左右、`AS_RightX`からの信号をもとに移動回転をVector計算して制御します。
* **デッドゾーン:** スティック値が**\-20〜\+20**の範囲内ではモータが停止します。

### 2. ピッチ調整 (`Pitch`)

* ボリューム (`AS_Vol`) の値に応じて射出角度を調整します。
* 角度範囲: 約 **0\u00b0** (`AS_Vol`=0) から約 **90\u00b0** (`AS_Vol`=255)。

### 3. ディスク射出シーケンス (`Shot`)

射出ボタン (`SW_SHOT`) が押されたときの自動シーケンスです。ただし、ローラーは事前に `Roller` シーケンスで ON された状態であることが前提です。

1. **ローラー起動の前提**: ローラーが回転状態（`RollerOnOff == 1`）かつ、ローラー起動から **1000 msec** 以上経過している必要があります。
2. **射出タイミング**: ローラーが回転中かつ安定している状態で、前回射出から **500 msec** 以上経過した際に、射出サーボを `shotangle` (160°) に動かします。
3. **リターン**: 射出位置動作開始から **500 msec** 経過した時点で、射出サーボを `waitangle` (30°) に戻します（ボタンの離しとは直接関係しません）。

### 4. ローラーON/OFF 切替 (`Roller`)

ローラーボタン (`SW_ROLLER`) を押すことでローラーの回転状態をトグル（ON/OFF 切り替え）します。

- **チャタリング対策**: ローラーボタンの連打を防止するため、**500 msec** のインターバルを設定しています。
- **動作**: ローラーは ON のとき **100% PWM** で回転し、OFF のとき **0% PWM** で停止します。

### 5. 操作禁止時の動作

`OperationReady`が0（操作禁止）の場合、以下の動作を行います:

* 全てのモータを停止 (`MotorAllOFF()`)。
* 射出サーボを待機位置に戻す (`waitangle`へ)。
* BLモータを停止させる(`ROLLER`へ0を入力)
* オートレフェリ信号1がONの場合はブザーが鳴動 (`Buzzer1ON()`)。

## タイムアウト処理 (`RxController`)

コントローラからのデータ受信が**300 msec**以上途絶えた場合、**ControllerTimeout**フラグが`true`となり、全ての`RxData`が0にリセットされます。これにより、ロボットは**操作禁止状態**（`OperationReady = 0`）に移行します。
